01 / The situation
The old system wasn't failing loudly. It was slowing everyone down quietly.
Arnifi's existing design system was under-scaled for a product that was growing fast. The colour palette had no proper hue and saturation progression, so when designers needed a shade that actually worked, they stepped outside the system to find one. Developers then received designs full of colours with no token equivalent. Every handoff became a negotiation.
The diagnosis: the system was too small for how the team actually worked. The fix wasn't discipline, it was capacity.
02 / The decision
Comprehensive over "good enough", the call I went back and forth on most.
The safe move is to ship a lean set of components and grow it later. I made the opposite bet: build every component by hand, with enough variants to cover the states, sizes, and edge cases a designer would realistically reach for, so exploring a design direction never means leaving the library.
The accordion alone carries 280 variants; alerts, 576. The payoff came later, in speed nobody had to think about.
03 / Colour
Five families, real hue and saturation progression.
Arnique anchors the brand, a signature blend of "Arnifi + Unique". Tang signals caution. Dusk and Dawn carry the surfaces. A semantic set of True Blue, Moss Green, Ruby, Plum, and Tang gives every state a name that resolves to a token step.
With 10–12 steps per family, designers stopped needing colours that don't exist, and every colour a dev receives has a token name. That's the fix.

05 / Typography
Inter for the interface. Lora for the voice.
Inter carries the product UI across six weights, from heading scales to body copy to meta labels. Lora provides a serif display scale for marketing moments, campaign headers, and hero sections, a separate grammar for when the product needs to speak rather than function.
Every size, weight, and line-height is a named text style, Lo-D1-500, He-H1-600, Pa-P1-400, so typography decisions compound into a searchable system, not a guessing game.

13 / Validation
Tested in real work, not a review meeting.
Instead of presenting the library for feedback, I put it into active use: fellow designers explored live design directions using only the components, and the library went to developers implementing real features. First versions exposed gaps that only usage finds. Variants were reworked, definitions tightened, and the library grew where real work demanded it.
The strongest validation was unplanned: the system spread beyond the product. Ad creatives, campaign banners, and marketing videos all started pulling from the same tokens and components, one visual language across every surface Arnifi shipped.
